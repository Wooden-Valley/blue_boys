# Instacart Client Insights

Учебный проект: интеллектуальный клиентский сервис на данных покупательского
поведения (датасет **Instacart**). По идентификатору клиента приложение получает
аналитику его покупок из веб-сервисов **Loginom**, показывает её в
многостраничном веб-интерфейсе (таблицы, метрики, диаграммы) и формирует понятные
персональные рекомендации с помощью большой языковой модели **Mistral**.

Стек: **Loginom** (low-code аналитика, REST) → **Flask** (веб-приложение) →
**Mistral** через `langchain` (рекомендации), с кэшированием ответов модели.
Диаграммы — **Chart.js**, публикация — облако **Amvera**.

## Аналитические сервисы

| Сервис | Что показывает |
|---|---|
| Топ-10 товаров | Товары, которые клиент заказывает чаще всего |
| Забытые товары | Привычные товары, которых давно не было в заказах |
| Ритм покупок | Как часто и когда клиент оформляет заказы |
| Сводка по клиенту | Ключевые метрики покупательского поведения |
| Отделы и категории | Из каких отделов состоит корзина клиента |

Четыре сервиса опубликованы как REST-сервисы Loginom, «Отделы и категории»
рассчитываются в приложении из истории покупок. Каждый результат сопровождается
текстовым комментарием Mistral.

## Запуск

```bash
python -m venv venv
venv\Scripts\activate            # Windows
pip install -r webapp/requirements.txt
```

Создайте файл `.env` в корне с ключом Mistral:

```
MISTRAL=ваш_api_ключ
```

Запуск приложения:

```bash
cd webapp
python app.py                    # http://localhost:8000
```

## Как это работает

1. **Авторизация** — клиент вводит `user_id`; приложение проверяет его наличие
   через Loginom-сервис `GetUserHistory`.
2. **Аналитика** — по выбранному сервису Loginom (или расчётом в приложении)
   формируются данные; они показываются таблицей, плитками-метриками и
   диаграммой Chart.js.
3. **Рекомендация** — подготовленный текст передаётся в Mistral с промптом,
   подобранным под конкретный сервис; модель возвращает дружелюбный комментарий.
4. **Кэширование** — ответ модели сохраняется в `webapp/llm_cache.json` по ключу
   `sha256(сервис + user_id + данные)`; при повторном запросе с теми же
   параметрами ответ берётся из кэша (в интерфейсе — отметка «из кэша» и время
   ответа), а не запрашивается у модели заново.

## Публикация

Приложение развёрнуто в облаке **Amvera**: <https://practica2-gekyme.amvera.io/>.
Порт читается из переменной окружения `PORT`, приложение слушает `0.0.0.0`,
секреты передаются через переменные окружения платформы.
